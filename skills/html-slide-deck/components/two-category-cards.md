# Two Category Cards

## 用途

表达两个对象、状态、阶段、方案或类别之间的并列、对比或转化关系。

## 适合

- 当前 / 目标
- 问题 / 动作
- 风险 / 机会
- 内部 / 外部
- 传统方案 / 新方案
- 错误理解 / 正确理解

## 输入参数

- `section_label`
- `title`
- `left.label`
- `left.title`
- `left.subtitle?`
- `left.points[]?`
- `left.visual?`
- `right.label`
- `right.title`
- `right.subtitle?`
- `right.points[]?`
- `right.visual?`
- `bottom_takeaway?`

## 稳定核心

- 页面主体必须是两个主对象。
- 两个对象应视觉可比较，除非内容本身有明显主次。
- 底部观点可以总结二者关系，但不能变成第三个平级对象。

## 密度规则

- 左右两侧都必须有有效内容或视觉锚点；任一侧有效高度低于主体区 `55%` 时，不要继续使用等高双大卡。
- 如果一侧内容明显少，改成 `visual-left-right-list`、大字观点 + 辅助解释、非对称图文，或把少的一侧变成标签/指标区。
- 不要用空白卡片撑满双栏。每侧至少应有标题 + 2 个信息点，或标题 + 1 个清晰图形/指标。
- 底部 takeaway 应贴近主体，不要在页面底部孤立漂浮。

## 常用变体

- `equal-cards`: 两边同等重要，每边标题加 1-3 个要点。
- `with-bottom-takeaway`: 上方两个分类，下方一条结论、原则或行动判断。
- `visual-left-right-list`: 一边截图、图示或大视觉，一边解释列表。
- `problem-action`: 左边讲问题或约束，右边讲动作或方案，右侧可更重。
- `before-after`: 左边现状，右边目标或变化后状态。
- `metric-compare`: 两边各自突出一个关键数字或指标。

## 可调整范围

- 比例可以是 `1:1`、`5:7`、`7:5`。
- 卡片可变成面板、栏、带状态模块、图文模块。
- 视觉可使用图标、截图、小图表、占位图或纯文字。
- 内容过密时拆成连续页，不要压缩到不可读。

## 不适合

- 三个以上平级对象。
- 单一路径流程。
- 纯数据大字报。
- 没有明确二元关系的普通列表。
- 两侧内容严重不均衡且无法视觉补偿的页面。
