# Management Rhythm

## 用途

表达周期、节奏、阶段、管理层级或复盘机制。

## 输入参数

- `section_label`
- `title`
- `cadences[]`
  - `period`
  - `title`
  - `description`
  - `review_object?`

## 稳定核心

- 每个模块都应包含周期或阶段名称、核心动作、检查对象。
- 模块之间应有清晰的节奏递进或层级关系。

## 常用变体

- `three-cadence-cards`: 日 / 周 / 月，或短 / 中 / 长。
- `layered-review`: 战略层 / 运营层 / 执行层。
- `stage-rhythm`: 阶段 1 / 阶段 2 / 阶段 3。

## 不适合

- 纯时间里程碑，优先用 `timeline`。
- 复杂流程，优先用 `process-flow`。
