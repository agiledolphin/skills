# Data Hero

## 用途

用 1-3 个关键数字、指标或比例支撑一个核心判断。适合内容较少但需要强视觉中心的页面。

## 输入参数

- `section_label`
- `title`
- `metrics[]`
  - `value`
  - `label`
  - `explain?`
- `takeaway?`

## 稳定核心

- 数字是页面主视觉，不要埋在普通卡片里。
- 每个数字必须有清楚含义，不能只是装饰性大字。
- 数据大字页可以留白，但留白必须围绕数字形成视觉张力。

## 密度规则

- 单页最多 3 个关键数字；超过 3 个改成表格、矩阵或拆页。
- 每个数字配 1 句解释即可，不要堆长段落。
- 数字组应占主体区主要面积，不能缩在页面一角。

## 不适合

- 没有真实数字或明确指标的内容。
- 需要解释复杂机制、流程或多维对比的内容。
